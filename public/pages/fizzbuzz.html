FizzBuzz (3 versions)

Browser version
Run within the browser, but included RequireJS to allow unit testing with "chai".
Server version
Run within the server, and all answers sent back to browser in a completed format.
Comm version
  1. browser continuously sends requests for each calculation. waiting for each answer. (browser sends: calc-request-1, calc-request-2, ... etc.)
  2. server receives request and processes the data, then sends back: number, fizz, buzz, or fizzbuzz. (server might respond in different order: calc-request-2-answer, calc-request-1-answer, ... etc)
  3. browser receives answer and send acknowledgement back to server
  4. some answers from server might be consist of both number-and-letter which will signal the browser to wait an estimated amount of time.
  5. browser re-organizes the answers, and sends new request for any of the missing pieces of data, or any requests that have exceeded the estimated response time the server gave.
  6. server processes new requests only if there is no same request being processed or in queue.
